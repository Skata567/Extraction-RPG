# Unity 검증 루프 스킬

이 스킬은 기능 구현, 버그 수정, 리팩토링 후 사용한다.
목표는 “코드를 작성했다”에서 멈추지 않고 Unity에서 실제로 깨질 수 있는 지점을 점검하는 것이다.

## 언제 사용할 것

- 기능 구현을 마친 뒤
- 버그 수정 후
- PR을 만들기 전
- 씬, 프리팹, ScriptableObject, ProjectSettings를 건드린 뒤
- 플레이어 이동, 전투, 아이템, 인벤토리, 탈출, 판매 흐름을 수정한 뒤

## 검증 단계

### 1. 변경 범위 확인

- 변경한 파일 목록 확인
- `.cs`, `.unity`, `.prefab`, `.asset`, `ProjectSettings` 변경 여부 확인
- 의도하지 않은 변경이 섞였는지 확인

### 2. Unity 컴파일 확인

- Unity Editor Console에서 컴파일 오류가 없는지 확인한다.
- CLI나 CI로 Unity 테스트를 실행할 수 있는 환경이면 해당 명령을 사용한다.
- 실행 환경이 없으면 검증하지 못했다고 명확히 말한다.

### 3. 자동 테스트

가능한 경우 아래를 우선한다.

- EditMode 테스트: 인벤토리 추가/제거, 판매 계산, 골드 증감, 데미지 계산
- PlayMode 테스트: 플레이어 피격, 적 사망, 아이템 획득, 탈출 처리

테스트가 아직 없다면, 새 로직 중 순수 C#으로 분리된 부분부터 테스트 후보로 제안한다.

### 4. Unity 설정 확인

아래 항목을 점검한다.

- Inspector 참조가 비어 있지 않은가?
- Prefab에 필요한 컴포넌트가 붙어 있는가?
- Collider2D, Rigidbody2D, IsTrigger 설정이 의도와 맞는가?
- Tag와 Layer 설정이 코드와 일치하는가?
- UI Text, Button, Panel 참조가 연결되어 있는가?
- ScriptableObject 에셋이 올바르게 연결되어 있는가?
- 씬 전환 시 Manager가 중복 생성되지 않는가?

### 5. v0.1 핵심 루프 수동 테스트

가능한 빌드 또는 에디터 플레이에서 아래 순서로 확인한다.

1. 플레이어가 이동한다.
2. 마우스 방향 조준이 된다.
3. 기본 공격이 나간다.
4. 적이 플레이어를 감지하고 추적한다.
5. 적이 플레이어를 공격한다.
6. 플레이어 공격으로 적 체력이 감소한다.
7. 적 사망 시 아이템이 드랍된다.
8. 아이템을 줍는다.
9. 인벤토리에 아이템이 들어간다.
10. 탈출구에서 상호작용한다.
11. 탈출 성공 시 아이템 판매 금액만큼 골드가 증가한다.
12. 다시 던전에 입장할 수 있다.
13. 사망 시 이번 원정 아이템이 사라진다.

### 6. 회귀 위험 확인

- 기존 입력키가 바뀌었는가?
- 기존 씬 오브젝트 이름이나 태그 의존성이 깨졌는가?
- 프리팹 Override가 의도치 않게 변경됐는가?
- ScriptableObject 기본값이 런타임 테스트 중 바뀌었는가?

## 검증 보고 형식

작업 후 아래 형식으로 정리한다.

```md
# 검증 결과

- 컴파일: 확인/미확인/실패
- 자동 테스트: 통과/미실행/실패
- 수동 테스트: 통과/부분 확인/미실행
- Unity 설정 확인: 완료/부분 확인/미확인

# 발견한 문제

# 남은 위험

# 다음 확인 사항
```

검증하지 못한 항목은 통과했다고 말하지 않는다.

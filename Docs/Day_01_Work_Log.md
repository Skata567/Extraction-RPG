## 오늘 수정한 주요 문제

### Unity 6 내장 폰트 오류 수정

발생한 오류:

```text
ArgumentException: Arial.ttf is no longer a valid built in font. Please use LegacyRuntime.ttf
```

원인:

- Unity 6에서는 `Resources.GetBuiltinResource<Font>("Arial.ttf")`를 더 이상 유효한 내장 폰트로 쓰지 않는다.

해결:

- 런타임 UI 생성 코드에서 `Arial.ttf` 대신 `LegacyRuntime.ttf`를 사용하도록 수정했다.

왜 필요한가:

- 런타임 HUD와 패널 UI가 생성되는 순간 예외가 발생하면 핵심 루프 테스트가 막히기 때문이다.

## 시간 소모 루팅/탈출 시스템

추가한 방향:

- E를 누르면 즉시 루팅/탈출되지 않는다.
- 일정 시간 동안 게이지가 찬다.
- 진행 중 범위를 벗어나거나 피격되면 취소된다.
- 완료되면 루팅 UI가 열리거나 탈출 성공이 처리된다.

추가/수정한 주요 개념:

- `IProgressInteractable`
- `LootContainer`
- `InteractionProgressUI`
- `LootPanelUI`
- `PlayerInteractor` 진행형 상호작용 처리
- `DungeonExit` 진행형 탈출 처리
- `ItemDropper`가 즉시 아이템 대신 루트 컨테이너 생성

왜 이렇게 만들었는가:

Extraction RPG의 핵심 긴장은 "위험한 장소에서 더 파밍할지, 안전하게 탈출할지"의 선택에서 나온다. 루팅과 탈출이 즉시 완료되면 이 긴장이 약해진다.

시간 소모 상호작용을 만들면 다음 시스템과 자연스럽게 연결된다.

- 루팅 중 적에게 들키는 소리 시스템
- 루팅 중 피격 취소
- 탈출 준비 중 방해받는 상황
- 무거운 상자일수록 긴 수색 시간
- 직업/장비/스탯에 따른 수색 시간 감소
- 특정 아이템을 찾는 동안 위험 감수

## Tarkov식 공통 인벤토리 UI v0.1

오늘 구현한 방향:

- 기존 회색 전리품 패널을 더 큰 공통 인벤토리 창 구조로 변경했다.
- 왼쪽은 플레이어 영역, 오른쪽은 대상 컨테이너 영역으로 나눴다.
- 장비 슬롯은 실제 기능 없이 자리만 표시한다.
- 전리품 아이템은 클릭 또는 드래그로 플레이어 가방에 옮길 수 있다.
- 아이템에 마우스를 올리면 툴팁이 표시된다.

왜 이렇게 만들었는가:

- 전리품, 가방, 상점, 창고를 각각 따로 만들면 UI와 입력 로직이 중복된다.
- 공통 창 구조를 먼저 잡으면 나중에 대상만 바꾸어서 재사용할 수 있다.
- 지금은 v0.1이므로 장비 슬롯 로직까지 한 번에 만들지 않고, 화면 자리만 잡았다.

현재 한계:

- UI가 아직 프리팹이 아니라 런타임 코드로 생성된다.
- 장비 슬롯은 실제 장착 기능과 연결되어 있지 않다.
- 상점/창고는 아직 같은 창 구조를 쓰도록 연결되어 있지 않다.
- 툴팁은 기본 정보만 보여준다.

## 앞으로 연결할 시스템

### 1. UI 프리팹화

현재 런타임 코드로 만드는 `LootInventoryWindow`를 프리팹으로 전환한다.

필요한 프리팹:

- `InventoryWindow_RT.prefab`
- `InventoryGrid_RT.prefab`
- `InventorySlot_RT.prefab`

목표:

- Unity Editor에서 위치, 색상, 폰트, 배경 이미지, 슬롯 이미지를 쉽게 수정한다.
- 코드에는 UI 생성보다 UI 연결과 동작만 남긴다.

### 2. 실제 장비 슬롯

현재 장비 슬롯은 자리 표시만 한다.

다음 단계:

- `EquipmentSlotUI`
- 슬롯 타입
- `PlayerEquipment`와 연결
- 무기 장착/해제
- 방어구 슬롯 추가

단, v0.1 핵심 루프가 안정된 뒤 진행한다.

### 3. 위험한 루팅 시스템

루팅/탈출 중 소리 이벤트를 발생시킨다.

예상 흐름:

- 루팅 시작
- 일정 반경에 소리 발생
- 근처 적이 조사 또는 추적 상태로 전환
- 플레이어가 계속 루팅할지 취소하고 도망칠지 선택

### 4. 상점/판매 UI 개선

현재 판매는 단순 정산 흐름에 가깝다.

다음 단계:

- 전리품 목록을 상점 창에서 확인
- 판매할 아이템 선택
- 전체 판매
- 아이템별 가격 표시

### 5. 창고/보관함

탈출 성공 후 아이템을 바로 판매하지 않고 보관할 수 있게 한다.

주의:

- v0.1에서는 과하게 만들지 않는다.
- 먼저 판매 루프가 안정된 뒤 확장한다.

## 오늘 결정한 설계 원칙

- 즉시 루팅보다 시간 소모 루팅을 기본으로 한다.
- 루팅 중 이동은 허용하지만 범위 이탈 시 취소한다.
- 루팅 중 피격되면 취소한다.
- 전리품 컨테이너와 플레이어 인벤토리는 소유 주체를 분리한다.
- 아이템 이동은 성공했을 때만 컨테이너에서 제거한다.
- UI 로직은 나중에 프리팹화할 수 있게 구성한다.
- 장비, 상점, 창고는 지금 당장 완성하지 않고 연결점만 만든다.

## Day 01 결론

오늘은 완성형 게임을 만든 것이 아니라, Extraction RPG의 핵심 감각인 "위험한 장소에서 시간을 들여 루팅하고 살아서 빠져나오는 구조"를 만들기 위한 기반을 세웠다.

다음 작업은 프리팹 기반 UI로 전환해서, 코드가 UI를 직접 그리는 구조를 줄이고 Unity Editor에서 수정 가능한 상태로 만드는 것이다.
